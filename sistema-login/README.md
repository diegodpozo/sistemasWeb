# SISTEMA DE GESTION DE HORARIOS Y LOGIN SEGURO

ESTE PROYECTO ES UN SISTEMA DE AUTENTICACION ROBUSTO DESARROLLADO EN PHP QUE INCLUYE UNA HERRAMIENTA DE GESTION DE HORARIOS DE MATERIAS. EL ENFOQUE PRINCIPAL ES LA SEGURIDAD Y UNA INTERFAZ MODERNA Y MINIMALISTA.

## INSTRUCCIONES DE INSTALACION

1.  **REQUISITOS PREVIOS**:
    *   SERVIDOR WEB (EJEMPLO: XAMPP, WAMP O LARAGON).
    *   PHP 7.4 O SUPERIOR.
    *   MOTOR DE BASE DE DATOS MYSQL.

2.  **CONFIGURACION DE LA BASE DE DATOS**:
    *   CREAR UNA BASE DE DATOS LLAMADA `esquema` EN TU SERVIDOR MYSQL.
    *   IMPORTAR EL ARCHIVO LOCALIZADO EN `sql/esquema.sql` PARA GENERAR LAS TABLAS NECESARIAS.

3.  **CONFIGURACION DEL PROYECTO**:
    *   COPIAR LA CARPETA DEL PROYECTO AL DIRECTORIO RAIZ DE TU SERVIDOR (EJEMPLO: `htdocs` EN XAMPP).
    *   VERIFICAR LAS CREDENCIALES DE CONEXION EN EL ARCHIVO `config/BaseDeDatos.php`.

4.  **ACCESO AL SISTEMA**:
    *   ABRIR EL NAVEGADOR Y DIRIGIRSE A `http://localhost/sistema-login/index.php`.
    *   SI NO TIENES UNA CUENTA, ACCEDER A LA OPCION DE REGISTRO.

## DIAGRAMA DE LA BASE DE DATOS

EL SISTEMA UTILIZA DOS TABLAS PRINCIPALES PARA LA GESTION DE ACCESO:

*   **TABLA `Usuarios`**:
    *   `id`: IDENTIFICADOR UNICO (CLAVE PRIMARIA).
    *   `email`: CORREO ELECTRONICO DEL USUARIO (UNICO).
    *   `claveHash`: CONTRASEÑA CIFRADA MEDIANTE BCRYPT.
    *   `creadoEn`: FECHA Y HORA DE REGISTRO.

*   **TABLA `IntentosLogin`**:
    *   `id`: IDENTIFICADOR DEL REGISTRO.
    *   `email`: CORREO RELACIONADO AL INTENTO.
    *   `direccionIp`: DIRECCION IP DESDE DONDE SE REALIZO EL INTENTO.
    *   `fechaIntento`: MARCA DE TIEMPO DEL INTENTO FALLIDO.

## MEDIDAS DE SEGURIDAD IMPLEMENTADAS

1.  **PROTECCION CONTRA SQL INJECTION**: SE UTILIZA **PDO (PHP DATA OBJECTS)** CON SENTENCIAS PREPARADAS PARA TODAS LAS CONSULTAS, EVITANDO LA MANIPULACION DE DATOS EN LA BASE DE DATOS.

2.  **PREVENCION DE XSS (CROSS-SITE SCRIPTING)**: TODAS LAS SALIDAS DE DATOS AL NAVEGADOR SON PROCESADAS CON `htmlspecialchars`, ASEGURANDO QUE EL CONTENIDO INGRESADO POR EL USUARIO SE MUESTRE COMO TEXTO PLANO Y NO COMO CODIGO EJECUTABLE.

3.  **VALIDACION CSRF (CROSS-SITE REQUEST FORGERY)**: SE GENERAN TOKENS UNICOS POR SESION QUE SE VALIDAN EN CADA ENVIO DE FORMULARIO PARA ASEGURAR QUE LAS PETICIONES SEAN LEGITIMAS Y SE ORIGINEN EN EL PROPIO SITIO.

4.  **GESTION SEGURA DE CONTRASEÑAS**: NUNCA SE ALMACENAN CLAVES EN TEXTO PLANO. SE UTILIZA EL ALGORITMO **BCRYPT** A TRAVES DE LA FUNCION `password_hash`, QUE ES EL ESTANDAR ACTUAL DE SEGURIDAD.

5.  **PROTECCION CONTRA FUERZA BRUTA**: EL SISTEMA REGISTRA LOS INTENTOS FALLIDOS Y BLOQUEA TEMPORALMENTE LA CUENTA TRAS **5 INTENTOS INCORRECTOS** DURANTE UN PERIODO DE 15 MINUTOS.

6.  **SEGURIDAD DE SESIONES**:
    *   SE CONFIGURAN COOKIES CON LOS ATRIBUTOS `HttpOnly` (PARA EVITAR ACCESO DESDE JS) Y `SameSite=Strict` (PARA EVITAR ATAQUES DE ORIGEN CRUZADO).
    *   SE REGENERA EL ID DE SESION EN CADA LOGIN EXITOSO PARA PREVENIR EL SECUESTRO DE SESION.

7.  **CABECERAS DE SEGURIDAD HTTP**: SE IMPLEMENTAN HEADERS COMO `Content-Security-Policy`, `X-Frame-Options: DENY` Y `X-Content-Type-Options: nosniff` PARA FORTALECER LA DEFENSA DEL NAVEGADOR.
